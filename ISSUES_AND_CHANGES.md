# 项目架构问题与改动方向

## 核心结论：从"质量门控"转向"体动分段 + 双线分析"

当前 pipeline 是质量门控——固定 30s 窗口 → SQI 打分 → 筛掉不可靠窗口 → 输出通过的呼吸率。这个方案在"只要呼吸率"的场景是完整的，但如果最终目标是呼吸事件检测（暂停/低通气），架构需要调整。

根本矛盾：SQI 筛掉的 `low_quality` 窗口里混入了两类截然不同的东西——体动污染（该扔）和呼吸静默（事件候选，不该扔）。

---

## 整体架构：体动分段 + 双线 + SQI 建基线

```
原始信号 (PVDF + PR)
  │
  ├─ 体动检测（逐采样点，三分量融合）→ motion_mask
  │
  ├─ 按体动分段 → [段1, 段2, 段3, ...]
  │    每段 = 两次体动之间的安静期
  │    体动是天然分界点——翻身时信号本身无呼吸信息，
  │    切在这里不会破坏事件的完整性
  │
  ├─ 段长 < 10s（最短事件阈值）→ 直接丢弃，不予分析
  │
  ├─────────────────────────────────────────────────
  │                    │                    │
  │  SQI 筛选         │  线路 A: 呼吸率     │  线路 B: 事件检测
  │                    │                    │
  │  在安静段内找      │  在安静段内找       │  在安静段内找
  │  "最稳定的呼吸"    │  "有呼吸的部分"     │  "没呼吸的部分"
  │                    │                    │
  │  30s 窗口 SQI ≥ 60 │  谷到谷分帧         │  低振幅连续区域
  │  + motion 最低     │  → RR 时间序列      │  + 持续 ≥ 10s
  │  → 正常基线        │  → 附带置信度       │  → 事件标记
  │  （RR、振幅）      │                    │  （类型、起止、时长）
  └─────────────────────────────────────────────────
```

## 为什么体动分段比固定窗口好

### 固定 30s 窗口的问题
```
固定窗口:  [0,30)  [5,35)  [10,40)  [15,45)  ...
                       ↑
                  一个 15s 的暂停被切成两半，跨 3 个窗口
```

### 体动分段
```
原始信号:  |···干净···| 体动 |······干净······| 体动 |·干净·| 体动 |······干净······|
           段1 (180s)       段2 (420s)              段3(85s)     段4 (600s)

段2 (420s):
  谷到谷分帧 → 帧1(3.8s), 帧2(3.5s), ..., 帧15(18s), 帧16(3.6s), ...
                                            ↑
                                    完整暂停事件，没被切断
                                               ↑
                                    谷间间隔 > 10s → 直接判暂停候选
```

### 优势
1. **事件不被切割**：体动之间是连续安静期，暂停完整保留
2. **每段基线稳定**：同段内呼吸幅度、心率一致，不跨段比较
3. **体动是天然边界**：翻身前后呼吸模式不连续，切在这里最合理
4. **段太短直接跳过**：两段体动之间 < 10s → 不够判事件 → 不分析

## SQI 的新角色：不参与事件判断，只管建基线

旧架构里 SQI 是核心输出——决定每个窗口的呼吸率能不能用。

新架构里体动分段在源头解决了"信号混杂"的问题——体动已经被切掉了，安静段内部不需要 SQI 来筛。但 SQI 仍然有用：

**建基线**：呼吸事件的定义依赖基线（低通气 = 振幅相对基线下降 ≥ 30%，暂停 = 下降 ≥ 90%）。SQI 筛选出最稳定的呼吸段来建立正常 RR 和振幅基线。

```
建基线:   用 SQI 最高的 N 个安静段 → 得到 正常RR, 正常振幅
线路 A:   所有安静段的 RR 时间序列（体动处断开）
线路 B:   相对基线的振幅下降 + 持续时间 → 事件
```

SQI 的底层特征在新架构里的去向：

| 特征 | 旧用途 | 新用途 |
|------|------|------|
| `motion_fraction` / `motion_score` | SQI 扣分 | 分段边界判断 |
| `resp_band_ratio` | SQI 加分 | 线路 B：低于阈值 + 持续 ≥ 10s → 暂停候选 |
| `dominance` | SQI 加分 | 线路 B：主峰消失 → 呼吸节律丧失 |
| `ibi_cv` | SQI 加分 | 线路 A：帧间稳定性 → RR 输出的置信度 |
| `rr_bpm` | SQI 输出 | 建基线（正常 RR）+ 线路 A 输出 |

## 呼吸周期分帧：谷到谷

固定 30s 窗口不适合事件检测，改用谷到谷分帧：

- 谷比峰干净（不受 BCG 叠加干扰），`find_peaks(sign=-1)` 已验证可行
- 谷间间隔 > 10s → 直接判呼吸暂停候选（纯规则）
- 事件检测场景 peak 失败回退 FFT 的逻辑要反过来——无峰即事件特征，不凑数
- 跳帧技巧：体动段置零或排除，`find_peaks` 只在干净段上跑，不被体动尖峰抢 prominence

## 小波能量 z-score 放大导致误判体动

**现象**：500s 附近被标记为体动，但 PVDF 呼吸波形肉眼无明显扰动，PR 通道纹丝不动。

**实测数据**：
```
体动触发点 501.64-502.12s（仅 0.48s），膨胀后覆盖 499.64-504.12s

三分量在该段:
  wavelet_score  = 13.25   ← 唯一过阈值的分量
  voltage_score  = 1.99    ← 不到 3.0
  envelope_score = 1.33    ← 不到 3.0

PVDF 绝对能量: 0.00028 V²，仅比干净区高 3.7 倍
PR std:        0.0015V，无任何变化
```

**根因**：`wavelet_motion_energy_z` 内部用全文件 `robust_positive_z` 做归一化。安静睡眠时高频细节能量的中位数和 MAD 极小，即使绝对能量只有微弱的 3.7 倍波动，z-score 也被放大到 13.25。0.2 × 13.25 = 2.65，加上 voltage 和 envelope 的小幅贡献后融合分数突破 3.0。这是统计学上的异常，生理上的正常。

**解决**：保留 PVDF 小波细节能量，但把它降级为“辅助证据”，默认对小波 z-score 做 `wavelet_z_cap = 5.0` 截断。这样默认权重 `wavelet_motion_weight = 0.20` 时，小波项最多贡献 1.0 分；只有 voltage/envelope 或 PR 也有明显变化时，融合分数才会越过体动阈值。500s 附近这种“PVDF 呼吸波形好、PR 不动、只有小波 z-score 虚高”的片段不再被判体动。

可调参数：

```powershell
python .\quality_gating.py --wavelet-z-cap 4 --out-dir .\outputs\wavelet_cap4
python .\quality_gating.py --wavelet-motion-weight 0 --out-dir .\outputs\no_wavelet
```

## 姿态变化导致 PVDF 呼吸幅度变化，低通气和体动容易误判

**现象**：侧睡、平躺或换位置后，PVDF 呼吸电压会整体变大或变小。肉眼看波形仍是规则呼吸，但旧逻辑可能把幅度下降判成低通气，或把较大的正常呼吸斜率/包络变化判成体动。

**判断**：当前阶段先不做低通气判断是合理的。PVDF 幅度不是稳定的潮气量代理量，它受人体姿态、接触位置、床垫耦合和传感器受力状态影响很大；没有多姿态标注数据时，用固定基线幅度直接判低通气，物理含义不稳。

**解决**：

- 默认开启 `motion_only_gate = True`：窗口通过与否只看坏点和体动，不再用呼吸频带、RR 合理性或 SQI 拒绝窗口。
- 默认关闭 `enable_event_detection`：`event_candidates.csv` 不再输出低通气/暂停候选；需要探索时手动加 `--detect-events`。
- 对体动候选加二次过滤：默认不再按持续时间删除候选段（`min_motion_burst_sec = 0s`），避免漏掉小体动；小/中等候选需要 PR 支撑 `pr_motion_support_threshold_z = 4.0`，或者 PVDF-only 扰动较强 `strong_pvdf_motion_threshold_z = 6.0`。

这样小体动能被保留；正常稍大的呼吸波形如果只是 PVDF 幅度变大、PR 不动、融合分数不高，仍不会被保留为体动。后续如果要做低通气/姿态分类，更适合上机器学习或至少先做人工作标注，把“姿态改变导致幅度变化”和“真实通气下降”分开。

## 是否需要机器学习

体动筛除当前不需要。一晚数据不够训练可靠模型，规则门控更可解释，也更适合先写小论文。

- 近期路线：先把体动片段筛准，输出干净段和窗口级体动指标。
- 低通气/姿态分类：等有更多姿态、位置变化和事件标注后，再考虑机器学习。
- 建议标注类别：正常平躺、正常侧睡、换位/翻身、离床/上床、疑似低通气、疑似暂停。
- 路线：规则门控 → 人工标注 → 看误差类型 → 再决定是否加轻量分类器。
